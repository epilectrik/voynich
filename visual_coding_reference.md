## Coding Decision Guide

### Root Features

**BRANCHING vs FIBROUS:**
- **BRANCHING:** 2+ distinct thick branches diverging from main root
- **FIBROUS:** Many thin, similar-sized roots without clear hierarchy

**BULBOUS:**
- Clearly swollen/rounded root structure (onion-like, turnip-like)

**SINGLE_TAPROOT:**
- Single main root going straight down

**When to use NONE:**
- No visible root structure in illustration
- Root clearly absent (cut plant, aerial portion only)

### Leaf Features

**LOBED vs COMPOUND:**
- **LOBED:** Single leaf with deep indentations (>25% into leaf body) but leaflets connected
- **COMPOUND:** Multiple separate leaflets attached to common stem

**SERRATED vs LOBED:**
- **SERRATED:** Shallow teeth (<25% into leaf margin)
- **LOBED:** Deep cuts (>25% into leaf)

**Leaf Arrangement:**
- **ALTERNATE:** Leaves attach singly, alternating sides of stem
- **OPPOSITE:** Leaves attach in pairs, facing each other
- **WHORLED:** 3+ leaves at same point on stem
- **BASAL:** Leaves arise from base of plant

### Flower Features

**Position:**
- **TERMINAL:** Flowers at stem tips
- **AXILLARY:** Flowers at leaf-stem junctions
- **THROUGHOUT:** Flowers distributed along stem

**Shape:**
- **SIMPLE:** Single layer of petals
- **COMPOUND:** Multiple layers or complex structure
- **RADIAL:** Symmetric around center point

### Complexity Scoring

| Level | Criteria |
|-------|----------|
| **SIMPLE** | <5 distinct visual elements |
| **MODERATE** | 5-15 distinct elements |
| **COMPLEX** | >15 distinct elements |

Elements include: individual leaves, flowers, roots, branches, labels

### When to Use UNDETERMINED

**USE UNDETERMINED when:**
- Physical damage obscures feature
- Artistic abstraction prevents clear identification
- Feature is genuinely ambiguous after careful examination

**DO NOT use UNDETERMINED for:**
- Uncertainty about identification (make best judgment)
- Features that are simply absent (use ABSENT or NONE)
- Preference to avoid decision (make a choice)

## Coding Workflow

### Recommended Order

1. **Calibration set first** (5 folios): f38r, f9v, f56r, f3v, f42r
2. **Hub entities** (if not in calibration): f10v
3. **Verify f90v2** - confirm plant illustration present
4. **Short entries** (remaining 8): Simple illustrations, build consistency
5. **Isolated early folios** (remaining 3): f2r, f4v, f5r, f5v
6. **Medium entries** (remaining 8): More detail
7. **Long entries** (remaining 5): Most complex

### Quality Control

**At end of session:**
- Re-code f38r (simplest) without looking at original coding
- Re-code f42r (most important) without looking at original coding
- Compare for consistency

**Target:** >80% agreement on re-coded folios
